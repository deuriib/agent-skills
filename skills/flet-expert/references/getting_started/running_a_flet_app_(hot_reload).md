"Getting startedRunning a Flet app (Hot Reload)\nRunning a Flet app (Hot Reload)\n\nFlet apps can be executed as either desktop or web applications using the flet run command. Doing so will start the app in a native OS window or a web browser, respectively, with hot reload enabled to view changes in real-time.\n\nDesktop app​\n\nTo run Flet app as a desktop app, use the following command:\n\nuv\npip\nuv run flet run\n\n\nWhen you run the command without any arguments, main.py script in the current directory will be executed, by default.\n\nIf you need to provide a different path, use the following command:\n\nuv\npip\nuv run flet run [script]\n\n\nWhere [script] is a relative (ex: counter.py) or absolute (ex: /Users/john/projects/flet-app/main.py) path to the Python script you want to run.\n\nThe app will be started in a native OS window:\n\nmacOS\n\nWindows\n\nWeb app​\n\nTo run Flet app as a web app, use the --web (or -w) option:\n\nuv\npip\nuv run flet run --web [script]\n\n\n\nA new browser window/tab will be opened and the app will be using a random TCP port:\n\nRunning Flet app as a web app\nWatching for changes​\n\nBy default, Flet will watch the script file that was run and reload the app whenever the contents of this file are modified+saved, but will not watch for changes in other files.\n\nTo modify this behavior, you can use one or more of these flet run options:\n\n-d or --directory to watch for changes in the [script]s directory only\n-r or --recursive to watch for changes in the [script]s directory and all sub-directories recursively\nEXAMPLE\nuv\npip\nuv run flet run --recursive [script]\n\nEdit this page"
